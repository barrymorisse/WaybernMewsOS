# mission.md — Waybern Mews OS

## What Is This?

Waybern Mews OS is a personal property management web application built for Barry Morisse, chairperson of the Waybern Mews body corporate in Johannesburg, South Africa. It is a private, single-user tool — not a commercial product.

The complex consists of 5 sectional title residential units. Barry is in the process of terminating the managing agent and taking full operational responsibility himself. This software exists to make that possible without creating an unmanageable personal workload.

---

## The Core Problem

Managing a body corporate involves a recurring set of administrative tasks: billing, communication, maintenance, compliance, financial reporting, and record-keeping. These tasks are not intellectually demanding, but they are time-consuming, repetitive, and stress-inducing when done manually across disconnected tools (email, spreadsheets, PDFs, memory).

Barry currently has no unified system. Work lives in his head, his inbox, and various files. There is no audit trail, no automation, and no single source of truth.

---

## The Mission

**To give Barry back his time.**

Waybern Mews OS should handle as much of the administrative burden of running the complex as possible — automatically where feasible, with minimal manual input where not. Every feature should be evaluated against one question: *does this meaningfully reduce the time and stress Barry spends managing Waybern Mews?*

The software is not trying to be a general property management platform. It is a focused personal tool that does exactly what Barry needs, nothing more.

---

## Guiding Principles

1. **Efficiency above all.** Every workflow should be as fast as possible for a single user who knows the system well. No unnecessary steps, no unnecessary screens.

2. **Automate the repetitive.** Anything that happens monthly, quarterly, or annually should be automated or semi-automated. Barry should only need to review and approve, not construct from scratch.

3. **One source of truth.** All data — units, owners, finances, maintenance, communications — lives in one place. No more hunting across files and inboxes.

4. **Offline first, cloud backup.** The app runs locally on Barry's MacBook. Data is backed up automatically to Google Drive. No dependency on internet connectivity for day-to-day use.

5. **Grow over time.** The system is designed as an operating system — a stable base that absorbs new modules as Barry takes on more responsibilities. New features should slot in without breaking what exists.

6. **Simple and honest.** The UI should be clean, minimal, and direct. It should reflect the state of the complex accurately and make problems visible rather than hiding them.

---

## Who Uses This?

**Barry Morisse only.** This is not a multi-user system. There are no roles, no permissions, no onboarding flows. The UX can assume full familiarity with the complex and full trust in the user.

---

## What Success Looks Like

In 6 months, Barry should be able to say:

- "I spend less than 2 hours per week on Waybern Mews administration."
- "I know exactly what is owed, what is outstanding, what needs attention."
- "I have a complete record of everything that has happened at the complex."
- "I don't feel stressed when I think about managing the complex."
